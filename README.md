# Бот для технической поддержки

Этот проект представляет собой ботов для Telegram и VKontakte. Боты анализируют полученные от пользователей сообщения и отвечают в соответствии с их запросом. Если бот не понимает вопрос - он не отвечает пользователю, вместо бота должен ответить оператор. Боты в своей работе используют Dialogflow (искусственный интеллект от Google).

## Что нужно для работы

1. Скачать и распаковать файлы проекта в любой удобное место.
2. [Python](https://www.python.org/downloads/) версии 3.10.11 (на версиях ниже или выше проект не тестировался).
3. Аккаунт Google и настроенный в нем проект.
4. Аккаунт Telegram и созданный в нем бот через @BotFather, есть подробный [гайд](https://way23.ru/%D1%80%D0%B5%D0%B3%D0%B8%D1%81%D1%82%D1%80%D0%B0%D1%86%D0%B8%D1%8F-%D0%B1%D0%BE%D1%82%D0%B0-%D0%B2-telegram.html) по настройке ботов в Telegram. Вам потребуется узнать токен Вашего бота.
5. Настроенный агент в DialogFlow.

## Настройка Dialogflow

1. Создайте проект в Google Cloud
- Перейдите на https://console.cloud.google.com/
- Войдите в свой аккаунт google
- Создайте новый проект, можно дать стандартное название `support-bot`
- Запомните ID проекта, он понадобится позже, будет примерно такой `support-bot-123456`

2. Включите Dialogflow API
- На главной странице проекта введите в поиске `Dialogflow API`:

<img width="600" alt="image" src="https://github.com/user-attachments/assets/900822b9-3a86-40f7-953e-84dd3f815930" />

- Выберите его и включите, нажав на кнопку `Enable`.

3. Создайте агента Dialogflow
- Перейдите на сайт https://dialogflow.cloud.google.com/
- Создайте нового агента указав:
  - Имя - `support_agent` или любое другое
  - Язык `Default language` - обязательно выберите `Russian - ru`, иначе бот не будет распознавать русскую речь.
  - Часовой пояс `Default time zone` - стандартный `(GMT+3:00) Europe/Moscow` или выберите любой подходящий.
  - В `Google project` выберите Ваш только что созданный проект.

4. Создайте ключ доступа к сервису Google
- Вернитесь на страницу Вашего проекта Google по ссылке https://console.cloud.google.com/
- В меню справа откройте раздел `APIs & Services` -> `Credentials`
- Нажмите на кнопку `Create credentials` и создайте сервисный аккаунт:

<img width="600" alt="image" src="https://github.com/user-attachments/assets/fb1f4e4b-cd52-449b-ae50-c226a84de5d1" />

- Зайдите в созданный сервисный аккаунт, перейдите в раздел `Keys` и нажмите кнопку `Add key`, выберите параметр `json` и нажмите `Create`:

<img width="600" alt="image" src="https://github.com/user-attachments/assets/3e599011-5e68-47d0-b977-06e70f003415" />

- Начнется загрузка json ключа, сохраните его в надежном месте и никому не передайте, благодаря этому файлу боты смогут получить доступ к Dialogflow.

- Также сервисному аккаунту нужно расширить права, переходим в раздел `Permissions` -> `Manage access` -> дать роль `Dialogflow API Admin`

## Настройка переменных окружений

В папке с проектом лежит файл `.env_template`, поменяйте его название на `.env`, откройте его блокнотом и заполните необходимые данные в соответствии с нижеуказанной информацией:

```
TG_BOT_TOKEN='токен_от_Telegram_бота'
VK_BOT_TOKEN='токен_от_VK_бота'
PROJECT_ID='ID_вашего_Google_проекта'
GOOGLE_API_KEY='путь_до_json_ключа_от_сервиса_Google'
```
> Значения указывайте в кавычках.

> Если json с ключом от Google сервисов лежит не в папке с проектом лучше указать полный путь до файла

## Установка зависимостей и настройка виртуального окружения

1. У Вас уже должен быть установлен python версии 3.10.11
2. Открываете консоль и вводите следующую команду:
```
cd путь_до_папки_с_проектом
```
3. Устанавливаем виртуальное окружение, введя в консоли команду:
```
python -m venv venv
```
4. Активируем виртуальное окружение:
```
venv\Scripts\activate # для Windows
source bin/activate # для Linux
```
5. Устанавливаем зависимости
```
pip install -r requirements.txt
```

## Заполнение базы вопросов и ответов

Осталось наполнить базу Dialogflow вопросами и ответами на них, чтобы Вашим ботам было что отвечать пользователям.

1. В проекте лежит шаблон базы данных вопросов и ответов - `db_questions_and_answer.json`, откройте его.
2. Добавьте свои вопросы и ответы в соответствии с шаблоном, например:
```json
{
  "Как изменить пароль": {
    "questions": [
      "как сменить пароль?",
      "забыл пароль, что делать",
      "хочу изменить пароль"
    ],
    "answer": "Чтобы изменить пароль, зайдите в настройки профиля и нажмите 'Сменить пароль'"
  },
  "Как восстановить доступ": {
    "questions": [
      "не могу войти",
      "потерял доступ",
      "заблокировали аккаунт"
    ],
    "answer": "Напишите в поддержку на почту support@glagoly.ru"
  }
}
```

Правила заполнения:

- Название интента (например, `"Как изменить пароль"`) - должно быть уникальным
- Вопросы - укажите несколько формулировок одного и того же вопроса. Лучше больше, т.к. чем больше будет формулировок указано, тем в большем количестве различных сообщений боты смогут уловить суть вопроса
- Ответ - один конкретный и четкий ответ

3. Теперь данные из файла необходимо подгрузить в Dialogflow. Для этого был создан скрипт - `upload_intent.py`, чтобы не делать это руками. Для подгрузки вопросов и ответа в Dialogflow, введите в консоли команду:
```
python upload_intent.py -i "Название_вашего_интента"
```
> Чтобы загрузить несколько интентов, выполните данную команду для каждого из них отдельно.

## Запуск ботов

Ботов необходимо запускать в отдельных терминалах, так как работают они независимо друг от друга.

1. Запустим Telegram бота. Откройте первую консоль и введите следующие команды по порядку:
```
cd путь_до_папки_с_проектом
venv\Scripts\activate # для Windows
source bin/activate # для Linux
python tg_bot.py
```
> Если запуск прошел успешно, Вы должны увидеть сообщение: `Бот запущен. Для его отключения нажмите Ctrl+C`

> Не закрывайте консоль, иначе бот перестанет работать.

2. Запусти VK бота. Откройте новую консоль и введите эти команды по порядку:
```
cd путь_до_папки_с_проектом
venv\Scripts\activate # для Windows
source bin/activate # для Linux
python vk_bot.py
```
> Скрипт сначала попробует авторизоваться в VK с помощью указанного Вами ключа, вы увидите сообщение `Авторизация в VK...`. Если авторизация пройдет успешно, Вы увидите сообщение `Авторизация прошла успешно, ожидание новых сообщений`, значит бот работает.

> Не закрывайте консоль, иначе бот перестанет работать.

## Проверка работы

Проверка Telegram бота:
1. Найдите своего бота в Telegram по имени или ID
2. Отправьте ему команду `/start` - бот должен поздороваться
3. Отправьте ему вопрос из базы - бот должен дать ответ в соответствии с Вашим вопросом.

Проверка VK бота:
1. Зайдите в Вашу группу VK
2. Напишите группе любое сообщение от своего лица
3. Бот должен ответить, если знает ответ. Если он не знает ответ на вопрос, он должен промолчать

## Логирование в проекте

В проекте настроено автоматическое логирование. Это помогает отлаживать ошибки, отслеживать действия ботов и контролировать работу. Если что-то сломается или Вам нужно будет узнать, что происходило в определенный момент времени, Вы всегда сможете обратиться к логам.

Логи выводятся в терминал, а также автоматически сохраняются в файл, файл сохраняется по пути: `папка_с_проектом/logs/logs.log`.
> Если вы ещё не запускали проект данной папки не будет, но как только Вы запустите одного из ботов создастся папка и файл с логами.

Пример лога:
```
INFO:vk_bot: 2026-05-02 21:00:47,443 | Новое сообщение для меня от пользователя 49558646: хочу работать
```

Расшифровка:

- `INFO` - уровень важности
- `vk_bot` - от какого бота запись (скрипт `upload_intent.py` также записывает логи)
- `2026-05-02 21:00:47,443` - дата и время события
- `Новое сообщение для меня от...` - само сообщение

Как настраивать логирование:
1. Откройте файл `logger_conf.py`
2. Найдите строку:
```python
def setup_logger(level=logging.INFO, name=__name__):
```

3. Замените `logging.INFO` на нужный уровень:
- `logging.DEBUG` - для отладки
- `logging.INFO` - для обычной работы
- `logging.ERROR` - только ошибки

Управление размером логов:
1. Найдите следующий кусок кода, который отвечает за параметры сохранения логов:
```python
file_handler = RotatingFileHandler(
    log_path / 'logs.log',
    maxBytes=5 * 1024 * 1024,
    backupCount=3,
    encoding='utf-8',
)
```
2. Стандартно в проекте размер одного файла с логами настроен на 5Mb, если вы хотите увеличить или уменьшить выделяемую память под логи, поменяйте эту строчку(для примера поменяем на 10Mb:
```
maxBytes=10 * 1024 * 1024,
```
3. Стандартно в проекте настроено сохранение не больше 3 файлов с логами, если лимит закончится, самые старые стираются. Можно увеличить например до 5 поменяв следующую строчку:
```
backupCount=5,
```

- `vk_bot.py` - бот для VKontakte

- `logger_conf.py` - скрипт с настройкой логирования проекта
- `db_questions_and_answer.json` - база данных с вопросами и ответами (шаблон)
